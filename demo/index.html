02 — Stack

Nine open-source services, deliberately chosen.

Every tool was selected because it appears in real Operational Intelligence job descriptions and production industrial systems — not because it was easy.

01 EMQX
Transport · MQTT broker

Receives MQTT telemetry from edge devices on port 1883. The same protocol used in container terminals, cranes, and most industrial IoT environments.

02 Apache Kafka
Streaming · Distributed log

Buffers and replays messages reliably. Decouples the broker from the database — if storage goes down, no telemetry is lost.

03 PostgreSQL
Storage · Data warehouse

Stores raw telemetry in the public schema and analytics models in the analytics schema. Industry-standard relational database.

04 dbt
Transformation · SQL models

Builds a star schema with staging views, dimensions, and fact tables — all version-controlled, tested, and documented in SQL.

05 Apache Airflow
Orchestration · DAG scheduler

Runs the dbt pipeline every hour automatically. Provides retries, monitoring, and observability for the entire transformation layer.

06 Grafana
Visualisation · Live dashboards

Connects directly to PostgreSQL. The de facto standard for industrial monitoring — used by manufacturing, infrastructure, and OI teams.

07 Supervisor
Process control · Service manager

Manages the Python pipeline scripts as background services with automatic restart on failure. Web dashboard for full process control.

08 Docker
Containerisation · Runtime

All seven backend services run as Docker containers on a shared factory-network bridge — isolated, reproducible, restart-policy enforced.

09 Python
Glue · Pipeline scripts

Bridges between layers — paho-mqtt for MQTT, kafka-python for streaming, psycopg2 for the database. Where the streaming pipeline lives.

03 — Repository

How the codebase is organised.

The project is structured by responsibility. Every folder contains its own README explaining what's inside, why it exists, and how it integrates with the rest of the system.

github.com/arcotkaran/iiot-oi-platform
Open on GitHub
simulator/
Edge device code — Python simulator for three industrial machines (CNC mill, conveyor, hydraulic press) publishing MQTT telemetry every second.
pipeline/
The two streaming scripts: mqtt_kafka_bridge.py moves messages from EMQX to Kafka; kafka_to_postgres.py persists them to the database.
dbt/
Analytics transformations — staging views, machine dimension table, hourly performance fact table. Schema tests for data quality.
airflow/
DAG definitions for orchestration. The factory_dbt_hourly DAG runs dbt run followed by dbt test every hour.
infrastructure/
Docker container definitions, Supervisor configurations, network architecture, port forwarding rules, and full startup procedure.
grafana/
Exported dashboard JSON containing the four production panels — live temperatures, machine status, hydraulic pressure, and hourly aggregates.
docs/
Architecture diagrams, the full project report (PDF), and screenshots of every dashboard in the system.

04 — Highlights

What this project demonstrates.

Beyond the technology, the project showed that production systems are built through deliberate decisions and patient debugging.

01
End-to-end thinking

Designed and implemented every layer from edge ingestion to live dashboards — including the messy interface points between systems where most data engineering work actually happens.

02
Production patterns

Decoupled message buffering, automated orchestration, schema testing, process supervision — the same architectural patterns used in industrial systems at scale.

03
Real debugging

Diagnosed and resolved Kafka dual-listener configuration, WSL2 networking quirks, Airflow's SQLite locking issue, and Docker container DNS — the work hiring managers want to see.
